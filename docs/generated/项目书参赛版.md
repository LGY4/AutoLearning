## 1. 项目概述

### 1.1 项目简介

AutoLearning V2 是一套面向高校学生与自主学习者的个性化智能学习平台。系统围绕“学什么、怎么学、怎么练、怎么改、怎么评”五个核心问题展开设计，以学生画像为基础，以多智能体协同为核心，以知识图谱、题库训练、错题复习、学情分析和讯飞多模态能力接入为支撑，构建面向真实学习过程的闭环式智能学习系统。

与传统在线学习系统相比，AutoLearning V2 的目标不是提供静态课程内容，也不是仅仅充当一个问答机器人，而是将学习资源生成、学习路径规划、练习组织、错题复盘、能力分析和数字人交互整合为统一平台，使系统能够围绕不同学生的知识水平、学习风格、兴趣方向和薄弱环节，动态生成更匹配的学习支持方案。

### 1.2 项目定位

本项目定位为 AI Native 个性化学习平台，强调以下三点：

- 用学生画像驱动学习内容适配，而非统一模板式输出。
- 用多智能体完成分工协作，而非单模型串行生成全部内容。
- 用学习数据反哺学习过程，而非只做一次性内容生成。

### 1.3 建设目标

- 构建面向学习全过程的个性化智能学习平台。
- 实现多智能体协同生成课程文档、思维导图、练习题、代码案例和学习路径。
- 建立题库训练、错题复习、自适应难度和学情分析闭环。
- 接入讯飞数字人、语音、视频和 PPT 能力，增强交互与展示形式。

## 2. 项目背景与问题分析

### 2.1 项目背景

当前高校课程学习和在线教育平台普遍存在以下问题：

- 学习资源供给固定化，难以适配不同学生的基础与目标。
- 学习过程割裂，资料、练习、错题、路径和总结彼此分散。
- AI 工具多停留在单轮问答层面，缺乏结构化学习支撑能力。

随着大模型技术的发展，AI 已具备较强的文本生成与知识解释能力，但若缺少任务拆分、上下文管理、数据记录和学习过程建模，系统依然难以真正服务于教学与自学场景。

### 2.2 现有方案不足

传统题库系统和学习平台的不足主要体现在：

- 内容静态，难以根据学生差异动态调整。
- 学习路径依赖人工设计，缺少结构化依赖分析。
- 错题本只记录错误，无法自动组织复习与强化训练。
- 学情反馈停留在简单统计，缺少知识点与能力维度分析。
- 语音、数字人、视频等能力多为展示型功能，未深度嵌入学习业务。

### 2.3 本项目拟解决的问题

AutoLearning V2 主要解决以下问题：

- 如何为不同学生生成更适配的学习资源。
- 如何通过多智能体提升资源生成效率和内容质量。
- 如何把路径、练习、错题、复习、分析串成完整学习闭环。
- 如何把讯飞数字人和语音能力嵌入真实学习过程。

## 3. 需求分析

### 3.1 目标用户

- 需要系统化学习专业课程内容的高校学生。
- 需要自主规划学习节奏的自学者。
- 需要题库训练、错题复盘和能力分析的学习用户。
- 需要生成教学辅助材料和演示内容的师生群体。

### 3.2 功能需求

结合当前源码实现，系统核心功能需求包括：

- 用户注册、登录、JWT 身份认证与角色管理。
- 学生画像创建、编辑、查询与可视化展示。
- 智能对话、流式答疑和对话历史记录。
- 多智能体协作生成课程文档、思维导图、练习题和代码案例。
- 学习路径生成、阶段进度跟踪与路径查询。
- 题目生成、答案提交、错题查询、错题总结与邮件发送。
- 基于 SM-2 算法的复习调度与变式题生成。
- 轻量级知识图谱查询、学习顺序获取与路径辅助推荐。
- 学情分析看板与知识点掌握度统计。
- 代码沙箱在线执行。
- 讯飞数字人、语音识别、语音合成、视频生成与 PPT 报告生成。

### 3.3 非功能需求

- 系统应支持流式输出，提高交互实时性。
- 系统应具备模块化扩展能力，便于新增 Agent 与新功能。
- 系统应具备基础安全控制能力，包括认证、权限和密码安全。
- 系统应支持本地部署与 Docker 部署。
- 系统应具备一定容错与降级能力，降低外部服务异常的影响。

## 4. 总体方案设计

### 4.1 设计思路

本项目采用“前端交互层 - 后端接口层 - 智能编排层 - 业务服务层 - 数据存储层”的分层方案，通过学生画像和共享上下文将各功能模块串联起来。

总体思路如下：

- 前端负责学习交互、资源展示、图谱可视化和数据分析呈现。
- 后端负责接口组织、权限控制和业务流程协调。
- 编排层负责多智能体注册、调度和事件流组织。
- 服务层负责画像、知识检索、难度调整、复习调度和讯飞能力接入。
- 数据层负责学习数据持久化和知识内容索引。

### 4.2 技术路线

- 前端：React 18、TypeScript、Vite、TailwindCSS、Recharts、Markmap、KityMinder。
- 后端：FastAPI、Uvicorn、SQLAlchemy、Pydantic。
- 认证：JWT、bcrypt。
- 主模型通道：DeepSeek。
- 备用模型通道：讯飞星火。
- 多模态能力：讯飞数字人、TTS、STT、视频生成、PPT。
- 数据存储：SQLite，Redis 可选。

## 5. 系统架构设计

### 5.1 总体架构

系统整体分为四个核心层次：

- 接入层：Web 前端页面、资源中心、学习工作台、学情分析页、知识图谱页、数字人交互页、管理后台。
- 编排层：Orchestrator 与 7 个专业 Agent。
- 服务层：LLM 客户端、画像服务、RAG 服务、知识图谱服务、错题复习服务、自适应难度服务、学情分析服务、讯飞服务集。
- 数据层：SQLite 业务数据库、轻量知识库、文件资源与本地静态资源。

### 5.2 前端架构

前端采用 React 单页应用架构，当前代码中共包含 17 个页面组件，覆盖首页、登录注册、学生画像、学习工作台、学习路径、题库、错题本、资源中心、知识图谱、学情分析、数字人交互、学习视频和管理后台等页面。

前端具备以下特点：

- 支持流式消息渲染。
- 支持图谱、导图和图表可视化。
- 支持代码在线运行。
- 对部分外部能力保留降级逻辑，以保证演示稳定性。

### 5.3 后端架构

后端基于 FastAPI 实现，当前共包含 23 个 API 模块和 76 个路由接口，主要覆盖认证、画像、对话、路径、题库、资源、多 Agent 流式协作、分析、知识图谱、沙箱和讯飞能力等模块。

### 5.4 多智能体协同架构

系统当前实现了 7 个专业 Agent：

- ProfileAgent：画像分析。
- DocAgent：课程文档生成。
- MindmapAgent：思维导图生成。
- QuestionAgent：练习题生成。
- CodeAgent：代码案例生成。
- PathAgent：学习路径规划。
- TutorAgent：智能答疑辅导。

所有 Agent 均基于统一抽象基类实现，并由 Orchestrator 统一注册与调度，形成清晰的职责边界。

### 5.5 编排机制

Orchestrator 支持：

- 单 Agent 执行。
- 多 Agent 并行执行。
- 多 Agent 串行执行。

在“全套学习资源生成”场景下，系统会先执行 ProfileAgent，完成学生画像分析与个性化上下文构建；随后并行执行文档、导图、题目和代码四类 Agent；最后统一汇总输出结果，并通过 SSE 流持续推送执行状态和中间内容。

### 5.6 流式事件机制

系统通过 SSE 推送以下关键事件：

- agent_start
- agent_thinking
- agent_chunk
- agent_complete
- agent_error
- orchestrator_plan
- orchestration_complete

这使用户不仅能看到最终结果，也能看到资源生成过程，提高系统可解释性和交互体验。

## 6. 关键功能设计与实现

### 6.1 用户认证与权限控制

系统支持用户注册、登录、登出和当前用户信息查询，采用 JWT 进行身份认证，采用 bcrypt 进行密码哈希存储，角色区分为 student 与 admin。该模块为学习功能和管理功能提供统一权限基础。

### 6.2 学生画像模块

学生画像是个性化学习的基础。系统当前支持记录以下信息：

- 基本信息：姓名、年龄、年级、学校、专业。
- 学习信息：学习风格、知识水平、学习目标。
- 扩展特征：兴趣点、薄弱点、优势点和维度数据。

画像不仅用于展示，还会直接影响后续资源生成和问答输出。

### 6.3 画像驱动的动态更新机制

系统在对话结束后会对会话文本进行结构化提取，并将兴趣、薄弱点、优势项和学习偏好增量更新到画像中。这样系统可以随着交互次数增加，持续修正对用户的理解，增强后续生成内容的适配性。

### 6.4 智能对话与个性化辅导

系统提供普通对话和流式对话两种模式。TutorAgent 在回答时会自动读取学生画像信息，根据用户当前水平和学习风格调整解释方式、术语深度和举例策略，使辅导内容更贴近不同学生的实际需求。

### 6.5 多智能体学习资源生成

系统支持多类型学习资源生成，已实现的资源包括：

- 课程文档。
- 思维导图。
- 练习题。
- 拓展阅读。
- 代码案例。
- 评估报告。

其中，多 Agent 流式协作重点覆盖文档、思维导图、练习题和代码案例四类核心资源。系统先完成画像分析，再并行生成多个结果，提升了资源供给效率，也使不同资源类型保持较清晰的职责分工。

### 6.6 学习路径规划

系统支持按学习主题生成分阶段学习路径，并记录当前阶段、总阶段数和学习状态。路径中包含阶段目标、学习内容、预期时长和里程碑，适合将零散学习转化为可执行的阶段计划。

### 6.7 知识图谱与学习顺序推荐

系统内置机器学习领域知识图谱，采用有向无环图表示知识点前置依赖关系。当前实现了约 20 个知识节点，并支持：

- 前置知识查询。
- 学习顺序生成。
- 相关知识点获取。
- Mermaid 代码导出。
- 图谱统计信息输出。

该模块增强了路径规划的结构化依据。

### 6.8 题库训练与答题记录

系统支持按学习路径阶段动态生成题目，并记录用户答案、正确性和作答时长。答题数据会进一步用于：

- 错题本生成。
- 复习调度。
- 自适应难度分析。
- 学情分析看板。

### 6.9 错题本与错题总结

系统支持错题列表获取、错题复习分析、错题总结报告生成与邮箱发送。与传统错题本相比，本项目不是单纯展示题目，而是进一步分析错误原因、知识点薄弱项和后续改进建议。

### 6.10 基于 SM-2 的复习调度

系统采用经典 SM-2 间隔重复算法，为知识点构建复习计划，动态调整：

- 复习间隔。
- 重复次数。
- 难度因子。
- 下次复习时间。

这一机制使错题训练具备长期记忆强化能力，而非停留在一次性纠错。

### 6.11 自适应难度引擎

系统根据用户在某知识点上的历史正确率、答题时长和样本数量，计算推荐难度。正确率高则提升难度，正确率低则降低难度，样本不足时降低推荐置信度。该模块有助于实现更加个性化的练习组织。

### 6.12 学情分析看板

系统可对学习过程数据进行聚合分析，当前包括：

- 总答题数与正确率。
- 学习时长估算。
- 知识点掌握热力图。
- 最近 30 天学习趋势。
- 薄弱维度分析。
- 学习路径进度。
- 对话统计。

前端通过图表、雷达图和热力图等方式进行可视化呈现，提升反馈直观性。

### 6.13 代码沙箱

系统提供 Python 在线执行沙箱，支持危险关键字拦截、超时控制、输出回传和错误提示。与代码案例模块结合后，可形成“生成 - 运行 - 验证”的学习体验。

### 6.14 轻量级 RAG 知识库

系统实现了轻量级知识库检索能力，采用内存分块与 n-gram 相似度检索方式，不依赖额外向量数据库，适用于轻量部署和教学实验场景。

### 6.15 讯飞多模态能力接入

系统已封装以下讯飞能力：

- 星火大模型备用通道。
- 讯飞数字人会话接口。
- 语音识别接口。
- 语音合成接口。
- 视频生成接口。
- PPT 报告生成接口。

需要说明的是，后端已具备真实调用逻辑；前端数字人页面为保证演示稳定性，仍保留了部分演示视频和降级逻辑。这种做法有助于提高比赛展示的可控性。

## 7. 数据设计

### 7.1 数据库概况

系统当前使用 SQLite 作为主数据库，核心业务表共 8 张：

- users
- student_profiles
- learning_paths
- questions
- answer_records
- learning_resources
- conversations
- review_schedules

### 7.2 核心实体说明

用户表用于认证与角色管理；学生画像表用于个性化输入；学习路径表用于阶段学习计划存储；题库与答题记录表用于练习数据沉淀；学习资源表用于生成内容管理；对话记录表用于辅导过程记录；复习调度表用于 SM-2 复习计划管理。

整体数据设计能够支撑本项目“生成、训练、反馈、分析、复习”的全链路业务闭环。

## 8. 核心技术与关键算法

### 8.1 SharedContext 共享上下文

系统通过 SharedContext 在多 Agent 之间共享用户标识、学习主题、学生画像、结果数据和中间记录，避免 Agent 之间直接相互依赖，降低耦合度。

### 8.2 双通道 LLM 调用策略

系统采用 DeepSeek 主通道与讯飞星火备通道的双通道调用方式，在主通道异常时自动降级，提高系统稳定性。

### 8.3 拓扑排序学习顺序算法

知识图谱模块基于前置依赖关系构建子图，并通过拓扑排序生成学习顺序，适合用于路径规划与知识结构组织。

### 8.4 SM-2 间隔重复算法

系统依据回答质量、重复次数和难度因子动态调整复习计划，提升长期记忆效果。

### 8.5 自适应难度规则引擎

系统依据正确率、答题速度和样本规模综合推荐题目难度，使练习更贴近学生真实水平。

### 8.6 轻量级 RAG 检索策略

系统采用文本分块与 n-gram 相似度计算完成轻量级知识检索，减少部署依赖，提升演示可行性。

## 10. 创新点与项目特色

### 10.1 多智能体分工协同

相较于单模型问答系统，本项目将画像分析、文档生成、导图生成、题目生成、代码案例生成、路径规划和智能辅导拆分为 7 个职责清晰的 Agent，并通过统一编排器组织协作，提升了系统结构清晰度和资源生成效率。

### 10.2 画像驱动的个性化闭环

系统将学生画像真正纳入资源生成、学习辅导和后续更新过程，不是静态存档，而是持续参与业务决策。

### 10.3 学习全过程建模

项目不是单一功能堆叠，而是完整覆盖：

- 学什么：学习路径与知识图谱。
- 怎么学：文档、导图、代码案例、拓展阅读。
- 怎么练：题库与难度调节。
- 怎么改：错题本、复习调度、变式题。
- 怎么看：学情分析与报告输出。

### 10.4 讯飞生态深度融合

## 11. 应用价值与推广前景

AutoLearning V2 已形成一个结构完整、功能成链、具备较强展示力和扩展性的智能学习平台。项目的核心竞争力不在单一 AI 功能，而在于通过学生画像、多智能体协同、知识图谱、题库训练、错题复习、学情分析和讯飞多模态能力，构建了较完整的个性化学习闭环。
