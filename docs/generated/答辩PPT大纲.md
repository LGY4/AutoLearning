# AutoLearning V2 答辩PPT大纲

## 第1页 封面

- 项目名称：AutoLearning V2
- 副标题：基于多智能体协同与讯飞生态的个性化智能学习平台
- 参赛队伍、学校、成员、指导教师
- 一句话定位：面向学习全过程的 AI 个性化学习平台

## 第2页 项目背景

- 当前在线学习平台资源供给静态化、学习过程割裂化
- 传统 AI 工具多停留在单轮问答层面
- 个性化学习、结构化辅导和持续反馈仍是核心痛点
- 引出本项目：让 AI 从“会回答”走向“会组织学习过程”

## 第3页 问题分析

- 学习内容难以适配不同学生基础
- 学习路径缺乏结构化规划
- 错题复盘缺少长期复习机制
- 学情分析缺少知识点和能力维度视角
- 多模态能力与学习业务脱节

## 第4页 项目目标

- 以学生画像驱动个性化资源生成
- 以多智能体协同完成学习任务分工
- 以路径、练习、错题、分析构建闭环
- 以讯飞数字人和语音能力增强交互体验

## 第5页 系统总体方案

- 前端交互层
- 后端接口层
- 多智能体编排层
- 业务服务层
- 数据存储层

说明系统如何从用户输入一路流转到资源生成、训练和分析结果输出。

## 第6页 技术架构

- 前端：React 18 + TypeScript + Vite + TailwindCSS
- 后端：FastAPI + SQLAlchemy + SQLite
- 编排层：Orchestrator + 7 个专业 Agent
- 大模型：DeepSeek 主通道 + 讯飞星火备通道
- 多模态：讯飞数字人、TTS、STT、视频生成、PPT

强调技术栈与项目目标之间的对应关系。

## 第7页 核心创新一 多智能体协同架构

- 7 个 Agent 职责划分
- ProfileAgent 先行分析画像
- Doc/Mindmap/Question/Code 并行生成
- TutorAgent 负责个性化辅导
- PathAgent 负责路径规划

可配一张 Agent 协作关系图。

## 第8页 核心创新二 学生画像驱动机制

- 画像字段构成
- 画像如何参与提示词构建
- 对话结束后如何自动提取并更新画像
- 画像如何影响资源难度、表达方式和学习建议

一句总结：画像不是展示用数据，而是系统决策输入。

## 第9页 核心创新三 学习闭环设计

- 路径规划：知道先学什么
- 资源生成：获得讲义、导图、代码和题目
- 练习训练：做题并记录结果
- 错题复习：SM-2 自动安排复习
- 学情分析：形成趋势和薄弱项反馈

强调项目不是单点功能，而是闭环系统。

## 第10页 关键功能演示一 学习资源生成

- 课程文档生成
- 思维导图生成
- 练习题生成
- 代码案例生成
- 多 Agent 流式执行过程展示

建议配资源中心截图和 Agent 协作面板截图。

## 第11页 关键功能演示二 学习路径与知识图谱

- 知识图谱节点与前置依赖
- 拓扑排序学习顺序
- 路径阶段化设计
- 用户可视化查看学习路线

建议展示知识图谱页面与学习路径页面。

## 第12页 关键功能演示三 题库、错题与复习

- 动态生成题目
- 提交答案并记录答题数据
- 错题本汇总
- 变式题生成
- 基于 SM-2 的复习调度

说明“从做错到复习”的数据链路。

## 第13页 关键功能演示四 学情分析

- 总答题数和正确率
- 学习趋势
- 知识点掌握热力图
- 薄弱维度雷达图
- 学习路径进度

建议突出“数据反哺学习过程”的价值。

## 第14页 关键功能演示五 讯飞多模态能力

- 数字人会话接口
- 语音识别与语音合成
- 视频生成
- PPT 报告生成

说明当前实现状态：

- 后端接口已接入
- 前端为保证演示稳定性保留了部分降级逻辑

## 第15页 核心算法与关键机制

- SharedContext 共享上下文
- 双通道 LLM 降级策略
- 拓扑排序学习顺序算法
- SM-2 间隔重复算法
- 自适应难度规则引擎
- 轻量级 RAG 检索

本页强调技术深度。

## 第16页 工程成果与实现规模

- 7 个专业 Agent
- 23 个 API 模块
- 76 个路由接口
- 17 个前端页面组件
- 8 张核心业务表
- 约 20 个知识图谱节点

强调项目不是概念设计，而是已有较完整工程实现。

## 第17页 项目优势总结

- 选题契合 AI 教育与个性化学习方向
- 架构清晰，具有多智能体辨识度
- 功能完整，具备学习闭环
- 多模态能力增强了交互与展示效果
- 具备后续推广与扩展价值

## 第18页 不足与改进方向

- 更多学科知识图谱有待扩展
- 向量化知识库能力可进一步升级
- 数字人前端全链路联调仍可继续加强

体现团队对项目现状和未来演进的清醒认知。

- 学生个性化自学平台
- 竞赛训练与专题学习支持

说明项目的落地与推广价值。
